# Spec — Báo cáo PAN BK SL (HĐ) — Bảng kê sản lượng & quyết toán cước

**Source file analyzed:** `PAN_BK_SL_T04.26_HD.xlsx` (also present at `docs/PAN- BK SL T04.26 (HD).xlsx`).

This is the monthly settlement deliverable that AMT Phúc Lộc sends to its customer **Công ty TNHH PAN HẢI AN**. The file shows the period **26/03/2026 → 25/04/2026** (Tháng 04/2026, but on the 26th-of-prev-month → 25th-of-current-month accounting cut, NOT calendar-month).

## Filename convention

`{CUST}_BK_SL_T{MM}.{YY}_HD.xlsx`

- `CUST` — customer code (`PAN` in this file)
- `BK` — Bảng Kê (statement)
- `SL` — Sản Lượng (volumes / trip detail)
- `T04.26` — Tháng 4 năm 2026 (April 2026)
- `HD` — Hợp Đồng (contract — i.e. priced from the contract pricing table)

## Sheets (in order)

| Sheet | State | Purpose | Generated by us? |
|---|---|---|---|
| `BKTT T4.26` | visible | **Bảng Kê Thanh Toán** — settlement summary, aggregated by route × cont type | **YES** |
| `SL T4.26` | visible | **Sản Lượng** — every container trip in the period, one row per container | **YES** |
| `Sheet1` | visible | Free-form working data (yard transfer log) | NO — internal scratch |
| `Sheet4` | visible | Pivot summary by HĐ description (sum chuyến / sum cước) | NO — Excel pivot output, not part of deliverable |
| `HĐơn` | visible | Invoice line breakdown (per-container `Nội dung HĐ` / chuyến / cước) — looks like the source for invoicing software | NO (out of scope for v1; see "Future work") |
| `Trucking (HD)` | visible | Contract pricing matrix (Pan-side) with multiple repricing columns. Columns AK–AM hold the **current contract unit prices** the customer expects | NO — sourced from our `Pricing` table |
| `Sheet2` | hidden | Number-to-words helper formulas for cell K58 of BKTT | NO — we render the words server-side |

The customer **only checks BKTT and SL**. The other 5 sheets are internal scratch / pivots that the original creator left in — we do NOT need to reproduce them. **v1 ships BKTT + SL only.**

### Per-container price split

Our `TripOrder.unit_price` is the **trip-level** price (one number for the whole trip, regardless of how many containers it carries). The customer's SL sheet wants **per-container price**. Strategy:

1. If a trip has 1 container, that container's price is `TripOrder.unit_price`.
2. If a trip has N containers and they are the **same** `work_type`, split equally: `unit_price / N`. (rounded to nearest VND)
3. If a trip has N containers of **mixed** `work_type` (e.g. one F20 + one F40), look up each container's unit price from the `Pricing` table for that client + work_type + route. Fallback to equal split if no pricing found.

This is documented as an **assumption** because the source file's per-container prices look like contract prices (e.g. 654871 = exact match to `Trucking (HD)` row "PAN HA – HAI AN" hàng 20'), implying the author re-prices each container against the contract. Our system can do the same when pricing rules exist.

## Period parameter

User picks a calendar month (e.g. April 2026). Server interprets the date range as:

- **start** = 26th of the previous month
- **end** = 25th of the selected month

This matches the customer's file. We expose `period_year`, `period_month` and OPTIONAL `start_date`/`end_date` overrides for the rare case the customer asks for a different cut.

The same convention is also used for `salary_period_configs` (already in our DB with `from_day=26, to_day=25`-ish) — share the helper `salary_service.get_salary_period_dates` (already exists at `app/services/salary_service.py`) when the user doesn't provide explicit start/end.

## Filename produced

`{CLIENT_CODE}_BK_SL_T{MM}.{YY}_HD.xlsx` — e.g. `PAN_BK_SL_T04.26_HD.xlsx`

## Access control

- Endpoint: `GET /api/v1/reports/customer-settlement/export?client_id=...&year=2026&month=4`
- Required role: `accountant` or `superadmin` (via `require_roles`)
- Accountants can pick any client; the report cannot leak data across customers (filtered by `client_id` server-side).
