**Execution Workflow (The Micro-Iteration Loop):**

**Phase 0: Scope Definition (Lead QA Engineer - Run Once)**
1. Read $OBJECTIVE
2. Generate a lightweight "Test Execution Checklist" breaking down the application into granular, single-role flows (e.g., "Dispatcher: Create Transport Manifest," "Accountant: Generate Invoice").
3. Select the first incomplete item on this checklist to begin Phase 1.

**Phase 1: Focused QA Testing (Lead QA Engineer)**
1. Authenticate into $HOST using ONLY the account relevant to the current checklist item.
2. Test ONLY the specific business flow and UI components associated with this single task.
3. **Hard Cap:** Stop testing immediately once you have identified a maximum of **2 bugs or UI issues**.
4. Generate a brief "Micro-Bug Report." If no bugs are found for this specific flow, mark the item as "Passed" on your checklist and restart Phase 1 with the next item.

**Phase 2: Micro-Patching (Senior Software Engineer)**
1. Review the current "Micro-Bug Report."
2. Formulate and apply the necessary code changes to fix ONLY the reported issues. Do not attempt large refactors.
3. Respond with a concise "Patch Report" detailing the specific files changed, and hand the process back.

**Phase 3: Verification & Advancement (Lead QA Engineer)**
1. Review the "Patch Report."
2. Retest the specific bugs that were fixed.
3. If the bugs are resolved, mark the current flow as "Passed," update the checklist, and loop back to Phase 1 for the next item. If bugs remain, generate a new "Micro-Bug Report" for the current flow and return to Phase 2.

**Exit Condition:**
Strictly loop Phases 1, 2, and 3 until every item on the "Test Execution Checklist" is marked as "Passed." Once the entire checklist is clear, output the "Final QA Sign-Off." Begin Phase 0 now.
